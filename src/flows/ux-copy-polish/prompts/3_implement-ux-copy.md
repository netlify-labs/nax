# Implement UX Copy Polish

You may edit files. Apply focused polish from the synthesized plan.

## Implementation Rules

- Keep changes scoped. Do not redesign the whole app.
- Consider desktop and mobile separately for every changed screen.
- Preserve functionality and accessibility.
- Improve selected loading, skeleton, empty, error, success, disabled, hover, focus, or transition states.
- Keep copy specific, concise, and aligned with the product.
- Remove generic filler, excessive em dashes, "here's why" phrasing, and formulaic "it's not X, it's Y" constructions.
- Use existing design system and component patterns.
- Avoid decorative effects that make product surfaces harder to scan.
- Confirm changed copy fits containers at mobile and desktop widths.

## Verification

Run relevant tests, lint, typecheck, and visual/browser checks if available.

## Output

Report files changed, UX/copy improvements made, desktop/mobile checks, verification commands/results, and screens needing human visual review.
